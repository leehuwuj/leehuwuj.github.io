    2. Cài đặt Apache Hive

    Ở phần này, tôi sẽ hướng dẫn các bạn cách cài đặt Hive tích hợp với YARN và sử dụng Spark là excution engine. Phần này tôi dựa trên hướng dẫn cài đặt của tôi và đồng nghiệp ở ACB.

    Yêu cầu:

    • Hadoop và Yarn đã được cài đặt thành công.
    • Spark đã được cài đặt thành công.

    Các bạn có thể xem thêm hướng dẫn cài đặt Hadoop, Yarn, Spark multiple cluster tại đây:

    • Hướng dẫn cài đặt Hadoop và YARN multiple cluster
    • Hướng dẫn cài đặt Spark multiple cluster với YARN

    Thông tin phiên bản:

    Trước hết, chúng ta cần phải xác định đúng phiên bản phù hợp với hệ thống hiện tại của bạn. Mỗi phiên bản Hive sẽ tương thích với các phiên bản Hadoop và Spark khác nhau. Thời điểm hiện tại (năm 2020), Hive chỉ hỗ trợ tương thích tốt với Hadoop 2 trở lên và Spark 2.

    • Hive 2.x tương thích với Hadoop 2.x.y
    • Hive 3.x tương thích với Hadoop 3.x.y

    Trong bài này, tôi sẽ sử dụng Hive phiên bản Hive 2.3.7.

    1. Tạo thư mục chứa Hive:
      Lưu ý: Hướng dẫn này sẽ cài Hive trên quyền user hadoop. Vì vậy bạn phải đảm phiên làm việc hiện tại đã được chuyển sang user Hadoop của bạn. Để chuyển sang user hadoop, chúng ta thực thi câu lệnh sau:
      su hadoop
      Tiếp thep, chúng ta sẽ tạo đường dẫn và phân quyền đường dẫn đó cho user hadoop. 
    cd /opt/ 
sudo mkdir hive 
sudo chown -R hadoop /opt/hive
    1. Tải Hive: 
    cd ~
wget -O hive.tgz https://downloads.apache.org/hive/hive-2.3.7/apache-hive-2.3.7-bin.tar.gz

    Cài đặt biến môi trường.

    Chỉnh sửa file cài đặt biến môi trường của bạn, tùy theo distro linux mà sẽ chỉnh sửa file cấu hình khác nhau, có thể là .bashrc hoặc .bash_profile, tôi cấu hình trên Ubuntu, vì vậy sẽ chỉnh sửa file .bashrc. Chúng ta tiến hành mở file .bashrc và thêm các dòng sau: 

    # Hive Environment Configuration
export HIVE_HOME=/opt/hive
export PATH=$HIVE_HOME/bin:$PATH

    Tạo đường dẫn cho Hive trên HDFS:

    Hive có thể sử dụng hệ thống file local của hệ điều hành đang chạy, tuy nhiên mục đích của chúng ta là cần cấu hình để Hive có thể sử dụng hệ thống file phân tán (HDFS).

    hdfs dfs -mkdir /tmp
hdfs dfs -mkdir -p /user/hive/warehouse
hdfs dfs -chmod g+w /tmp
hdfs dfs -chmod g+w /user/hive/warehouse

    Hive sẽ cần một cơ sở dữ liệu để lưu trữ thông tin metastore của mình, mặc định Hive single node sử dụng Apache Derby chỉ cho mục đích thử nghiệm nên chúng ta cần sử dụng một database khác. MySQL và Postgres được khuyến nghị từ tài liệu của Hive nên trong bài này, tôi sẽ hướng dẫn các bạn cài đặt metastore trên PostgreSQL

    1. Cài đặt và cấu hình Postgres
    Việc đầu tiên là chúng ta cần phải cài đặt Postgres. Nếu bạn đã cài đặt Postgres trước đó thì phải đảm bảo các cấu hình về security ở phần dưới đã được câu hình đúng. Chúng ta cũng sẽ cần một user và database cho Hive để tránh xảy ra tình trạng xung đột tài nguyên.
    Chạy câu lệnh sau để cài đặt Postgres database:

    sudo apt-get install postgresql

    Tiếp theo chúng ta cần cấu hình phần kết nối tới postgres. Bạn phải đảm bảo kết nối tới postgres được cho phép, để kiểm tra chúng ta cần mở file postgresql.conf:
    sudo nano /etc/postgresql/10/main/postgresql.conf
    Kiểm tra listen_addresses được chỉ đúng địa chỉ của client. Nếu hệ thống không yêu cầu bảo mật cao, bạn có thể cấu hình như sau:

    listen_addresses = '*'
standard_conforming_strings      off

    Tiếp thep, chúng ta mở file pg_hba.conf và cấu hình như sau:
    sudo nano /etc/postgresql/10/main/pg_hba.conf
    host all all 0.0.0.0 0.0.0.0 md5
    2. Cài đặt driver và thêm postgresql jdbc driver cho Hive.
    Hive sẽ cần tới JDBC để kết nối tới Postgresql:

    sudo apt-get install libpostgresql-jdbc-java
ln -s /usr/share/java/postgresql-jdbc4.jar /opt/hive/lib/postgresql-jdbc4.jar

    3. Tạo user hive trên postgresql:
    Đăng nhập vào postgresql:
    sudo -u postgres psql
    Tạo user:

    postgres=# CREATE USER hive WITH PASSWORD '123123';
postgres=# CREATE DATABASE metastore;
